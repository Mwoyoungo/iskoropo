Tech Stack Decision
Mobile: Flutter (single codebase for iOS + Android)
Backend: Node.js + Supabase (auth, DB, realtime, storage)
AI: Gemini API — intake parsing, smart matching, style descriptions
Payments: Paystack (SA-first, supports ZAR, instant EFT)
Maps: Google Maps SDK (Flutter)
Notifications: Firebase Cloud Messaging
Hosting: Supabase Edge Functions for serverless API
Build Order
Start with Phase 1 + Phase 5 auth + Phase 5 payments in parallel — you cannot ship laundry without auth and payment.

Phase 2 (provider side) must ship before any real users — providers need to be onboarded first.

Phases 3 and 4 (barber, car wash) only start once laundry is live and generating real feedback.
